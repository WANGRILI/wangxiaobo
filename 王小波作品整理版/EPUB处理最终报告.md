## 技术细节记录

### EPUB文件结构特点

**观察到的共同问题**:
1. **元数据类型不一致**: 某些DC元数据使用tuple而非str
2. **Spine结构异常**: 阅读顺序中项目格式不符合规范
3. **命名空间问题**: 部分文件使用非标准命名空间
4. **内容编码问题**: HTML文件可能使用混合编码

**典型错误模式**:
```
TypeError: sequence item 0: expected str instance, tuple found
```

### 修复方法详解

#### 方法1: 元数据重新标准化
- **操作对象**: title, creator, language等DC元数据
- **修复策略**: 
  1. 提取有效字符串值（忽略空tuple）
  2. 重新添加标准化元数据
  3. 确保所有必需元数据完整

#### 方法2: 结构重新打包
- **操作对象**: 整个EPUB的HTML内容和资源
- **修复策略**:
  1. 重新创建EPUB对象
  2. 逐个添加HTML文件作为章节
  3. 重建标准目录结构（NCX + Nav）
  4. 添加完整的Spine

#### 方法3: 容器结构修复
- **操作对象**: container.xml, OPF, NCX等
- **修复策略**:
  1. 验证container.xml指向正确的OPF
  2. 修复OPF中的manifest引用
  3. 重建NCX目录结构

### TXT文件处理策略

#### 编码处理流程
1. **优先尝试编码顺序**: UTF-8 → GBK → GB2312 → ISO-8859-1
2. **回退机制**: 编码错误时尝试下一个编码
3. **错误容忍**: 最后使用ignore模式确保可读性
4. **编码记录**: 记录每个文件成功使用的编码
